# QuizApp 刷题练习工具

本机优先的刷题与笔记工具，支持内置题库、导入 JSON、分层练习、顺序练习、随机练习、背题模式、答案表、错题集、学习统计、本地回收站、AI 题目分析、AI 对话和题目绑定笔记。

## 题库目录与分类规则

默认题库文件统一放在 `./data/`。新版推荐按 `./data/科目/章节.json` 组织，例如 `./data/毛概/第一章.json`；旧版平铺在 `./data/毛概-第一章.json` 的文件仍然兼容。

应用显示分类优先靠 JSON 内部字段和应用本地层级配置完成，物理文件夹只是更清楚的存放方式：

- `subject`：第一级科目，例如 `毛概`、`习思`。
- `chapter`：第二级章节，例如 `导论`、`第一章`。
- `path`：可选的完整层级数组，例如 `["毛概", "导论"]`。当前内置题库主要使用两层；以后需要更多层级时继续扩展这个字段。
- `name`：题库显示名，也会作为字段缺失时的兜底来源。

加载优先级：

1. 优先读取 JSON 内的 `path`。
2. 没有 `path` 时读取 `subject` + `chapter`。
3. 仍然缺失时，从 `name` 或文件名按 `科目-章节` 解析。
4. 解析失败时归入 `未分类 / 综合练习`。

推荐新版目录：

```text
./data/
  毛概/
    导论.json
    第一章.json
  习思/
    导论.json
    第一章.json
```

旧版平铺目录也能继续读取：

```text
./data/
  毛概-导论.json
  毛概-第一章.json
```

应用会读取每个 JSON 内部的 `subject`、`chapter` 或 `path`，再在首页归到对应科目和章节。真实文件夹用于管理文件，不作为唯一分类依据。

## JSON 格式示例

```json
{
  "name": "毛概-导论",
  "subject": "毛概",
  "chapter": "导论",
  "path": ["毛概", "导论"],
  "questions": [
    {
      "id": "q1",
      "type": "单选",
      "q": "题干",
      "options": ["A选项", "B选项", "C选项", "D选项"],
      "ans": "C",
      "exp": "解析内容"
    }
  ]
}
```

字段说明：

- `questions`：题目数组。
- `q`：题干。
- `options`：选项数组。
- `ans`：正确答案，单选可用 `A`、`B`、`C`、`D`；多选可用类似 `AC`。
- `exp`：解析，可为空。
- `questionImages`：题目图片数组，支持 HTTPS 地址和图片 Data URL。
- `explanations.builtin`：题库来源自带的文字、图片或视频解析；界面显示为“内置答案与解析”。
- `subjective`：填空、解答、例题等主观题类型，可在平板手写工作台作答。

AI 题目分析不写入题库 JSON，始终与 `explanations.builtin` 分开保存在本机。

## 大型题库与笔记工作台

含大量 Base64 图片的题库超过本地存储阈值后会自动保存到 IndexedDB，`localStorage` 只保留题库索引。文件选择器支持一次选择多个 JSON，批量导入后统一保存，避免逐章导入时反复序列化全部题库。

平板横屏首页和科目工具区会显示“手写练题”。普通刷题页右上角也有手写入口，进入后绑定当前题目，返回时回到原来的题目和进度。

HTML/WebView 首页提供“自由笔记”入口，先进入全屏笔记资料库。资料库左侧按“全部、未分类、题库科目、自定义文件夹”组织笔记；题库科目自动映射为只读根文件夹，自定义文件夹支持一层新建、重命名和删除，删除文件夹时其中笔记回到“未分类”。题目笔记始终跟随绑定题目的科目，不能脱离科目移动。

笔记工作台采用从 [SpeedyNote](https://github.com/alpha-liu-01/SpeedyNote) 移植的文档模型，派生代码在 `notebook/speedynote-notebook.js`：

- `Document -> Page -> VectorLayer -> VectorStroke / InsertedObject` 分层存储。
- 当前文档 schema 为 5：文档增加 `folderId`、`cover`、`lastEditorMode`，页面增加纯文本正文 `bodyText`；schema 4 文档在读取时补默认值，不做破坏性批量迁移。
- 每道题有独立笔记本；自由笔记本不要求绑定题目。
- 桌面和平板横屏默认使用封面网格，手机默认使用紧凑列表，两端显示偏好分别保存；封面可选内置模板、本地裁切图片、第一页缩略图或恢复默认。
- 手机首次打开自由笔记默认进入整页文字模式并自动保存，可切回手写后把文字作为只读底层继续叠加笔迹；每个文档会记住最后使用的文字/手写模式。
- 手机手写界面只常驻单层顶部栏和底部书写工具栏；笔记目录与解析使用侧边抽屉，页面、笔触设置和更多操作使用底部面板，避免挤占画布。
- 支持分页、图层显示/锁定/排序、压感钢笔、荧光笔、笔画橡皮、撤销重做、双指缩放和平移。“笔写字”模式下手指平移、手写笔绘制；“手指写字”模式下手指和手写笔都可绘制。
- 选择工具支持点选、空白区域框选、多选合并拖动、复制粘贴、创建副本和组删除；旋转对象按真实旋转边界命中。
- 页面条显示真实笔迹/对象缩略图，可拖动排序、复制、新建和删除页面。
- 可离线导入 PDF 指定页码，PDF.js 将页面渲染为独立底图资源，笔迹和对象继续保存在上层；支持按 PDF 提取文本搜索笔记。
- 可导出当前页 PNG 或带 PDF 底图、可见手写图层和插入对象的整本多页 PDF；手机和平板使用系统文件保存器选择位置。
- Markdown 和 KaTeX 公式会保留原始源内容，可以再次编辑、拖动、复制、全文搜索并随 PDF 导出；Markdown 中的原始 HTML 不会执行。
- 每页可添加书签，书签可重命名、定位和搜索；删除页面时对应书签同步删除，撤销会一并恢复。
- 笔记库可打开知识图谱，以节点和连线展示笔记、标签、绑定/关联题目和 HTTPS 外部资料；支持搜索、缩放、拖动、重排和节点跳转。
- 笔记“信息”页支持标签以及题目、其他笔记和 HTTPS 外部资料关联；全文索引会同时检索标签、链接、文本对象和 PDF 原文。
- 右侧区分题库内置答案与解析、AI 分析和图层，不会用 AI 内容覆盖题库解析。
- 笔记保存到 IndexedDB 的 `notebooks` 对象仓库；PDF 页面底图和自选封面保存在 `notebook_assets`，不会进入每次撤销/重做的文档快照，并会随删除、SNBX 导入导出和完整备份清理或重映射。自定义文件夹保存在 `quizapp_notebook_folders_v1`；旧版 `ink_notes` 笔迹在首次打开对应题目时迁移，不删除旧记录。

笔迹平滑使用 vendored `perfect-freehand 1.2.3`，PDF 导入使用 `pdfjs-dist 3.11.174`，PDF 导出使用 `pdf-lib 1.17.1`，富文本使用 `marked 15.0.12` + `KaTeX 0.16.22` + `html2canvas 1.4.1`，知识图谱使用 `Cytoscape.js 3.33.1`。APK 和 EXE 构建会同时打包 `notebook/` 与 `vendor/`，不依赖 CDN。

27考研题库包的公开数据来源、边界、开源手写项目对比和后续路线见 [题库来源与手写学习平台路线图](docs/题库来源与手写学习平台路线图.md)。

## 本地编辑与回收站

用户在前端创建层级、重命名、移动题库时，不要求真实移动 `./data/` 里的文件。应用会把本地覆盖信息保存在浏览器本地存储中：

- `quizapp_library_nodes`：自定义层级。
- `quizapp_bank_meta`：题库显示名、路径、隐藏状态。
- `quizapp_banks`：用户导入题库。
- `quizapp_recycle_bin`：回收站数据。

设置页显示的默认题库目录是相对路径 `./data/`，回收站索引目录是 `./data/.quizapp-recycle/`。单文件浏览器运行时，真正可回档的数据保存在本地存储里；APK 内置题库删除时只做本机隐藏，不会物理删除 APK 内置资源。

原生版的“打开目录”按平台分别处理：桌面端打开应用自己的本地题库管理目录；Android 端在共享存储根目录创建 `QuizApp` 文件夹，与 `Tencent` 等根目录文件夹同级，并直接请求系统文件管理器打开它。Android 11 及以上版本需要先授予“所有文件访问”权限；无法直达时只退回共享存储根目录，不会打开 JSON 文件选择器。应用界面不展示设备绝对路径。

## 完整本地备份与恢复

原生版设置页可以导出单个 `.quizbackup v2` 文件，内容包括：

- SQLite 一致性快照：题库索引、层级、练习与答案状态、错题集、FSRS 复习、考试、学习统计和公告状态。
- QSettings 中的主题、界面和行为设置。
- 应用数据目录中的题目笔记、自由笔记和媒体 blob。
- 共享存储中的 `QuestionBanks`、`Notes` 和 `RecycleBin`。

备份采用流式二进制容器，不会把大型题库或笔记整体读入内存。包头清单记录格式版本、应用版本、构建提交、平台、数据计数、每个逻辑相对路径的大小和 SHA-256。数据库使用 `VACUUM INTO` 创建一致性快照；WAL/SHM、SAF 导入导出暂存、笔记删除暂存、临时恢复目录和 `Backups` 本身不会递归写入备份。

恢复时先完整校验并解压到应用私有暂存目录，显示版本、时间、大小和题库/题目/练习/考试/笔记数量，用户确认后等待重启。应用会在下次启动、打开数据库前替换数据库、设置、笔记和共享目录；任一写入失败都会回滚到恢复前状态。

API Key 默认从备份中删除；只有用户主动勾选后才会以明文写入。不含密钥的备份恢复时保留当前设备已有密钥。Windows 使用标准文件对话框；Android 使用系统“另存为”和“打开文档”选择位置，不要求为了备份授予全盘文件权限。备份始终保存在用户选择的本机位置，不会上传到服务器。

## 打包与导入约定

- 内置题库放在 `./data/`，方便 APK 和 EXE 打包脚本按相对路径收集。
- 用户导入 JSON 时，可以选择任意本地文件；导入后分类仍按 JSON 内的 `path`、`subject`、`chapter` 解析。
- 如果一个 JSON 没有写分类字段，建议至少把文件名写成 `科目-章节.json`，例如 `毛概-第一章.json`。

## AI 分析与对话

AI 接口使用 OpenAI 兼容的 `/chat/completions` 和 `/models`。默认提供 DeepSeek 预设并自动填写 `https://api.deepseek.com` 与 `deepseek-chat`，也可以切换到自定义 OpenAI 兼容地址。原生版 API Key 在 Windows 由 DPAPI 加密，在 Android 由 Android Keystore 的 AES-GCM 密钥加密；设置文件只保存密文或非敏感参数。旧 Web 版仍把 AI 配置和记录保存在当前浏览器或 WebView 本地数据中。两种版本都不会把密钥写进题库 JSON、APK、EXE 或 Git 仓库。

做题页的“AI 分析”会发送以下结构化内容：

- 题型、题干、科目和章节。
- 全部选项及选项字母。
- 正确答案字母、显示答案和对应的正确选项文本。
- 题库已有解析。

分析提示词要求模型依次说明结论、正确选项、错误选项、知识点、易错点和记忆提示。如果题目、选项与题库答案明显矛盾，模型应指出矛盾，但不能擅自改写题库给出的正确答案。每道题的分析结果按题目唯一键保存在本机，重新进入时默认收起；点击“追问”会新建对话，并把题目结构和第一次分析作为固定上下文。

AI 对话支持文本文件和图片附件。文本文件正文最多读取 30000 字符；图片最大 8MB。用户需要在 AI 设置中开启“模型支持图片”，图片才会按 `image_url` 视觉消息格式发送；未开启、附件数据已失效或文件正文无法提取时，内置提示词要求模型明确说明不能读取，不能假装看到内容。为避免本地存储过大，图片二进制不会随历史记录长期保存，对话中只持久化图片名称和类型。

AI 专属设置包括 API 地址、API Key、模型列表、最大输出 tokens、读取历史消息数、温度、视觉能力、追加系统提示词和本地对话保存开关。原生版可以直接读取模型列表并发送最小非流式请求测试连接；重定向不会自动携带密钥，远程地址必须使用 HTTPS，仅 `localhost` 和 `127.0.0.1` 调试地址允许 HTTP。旧 Web 版自定义 API 还需要允许浏览器或 WebView 跨域请求。

除基础题目分析外，当前版本还支持生成结构化相似题、归纳错因和总结当前笔记页。相似题以可交互题卡显示，答案默认隐藏；错因建议不会修改用户手动标签或备注；笔记摘要不会修改原笔记。以上学习任务统一保存在独立的 `quizapp_ai_learning_tasks_v1` 本地存储中，并记录模型、生成时间、用量和来源哈希；题目或笔记变化后会提示结果可能已经过期。

图片能力采用严格检查：依赖图片才能理解的题目不能发送给文本模型，含手写笔迹的笔记页也必须由用户确认当前模型真实支持图片后才能生成摘要；纯文本笔记仍可使用文本模型。即使启用视觉能力，AI 结果也始终与 `explanations.builtin` 内置答案和解析分开显示、分开保存。

## FSRS 复习队列

任意题目都可以通过“加入复习”创建复习卡。复习队列和错题集相互独立：错题集仍由用户手动维护，复习队列使用 [ts-fsrs](https://github.com/open-spaced-repetition/ts-fsrs) 5.4.1 根据“忘记、困难、良好、简单”四档评分计算下一次到期时间。

复习卡只保存题目唯一键、必要的文本快照和调度数据，不复制 Base64 题目图片。打开复习时优先从当前题库解析最新题目；原题库已删除时再使用文本快照。调度数据保存在本机 `quizapp_fsrs_review_cards`，不会上传。

## 模拟考试

“学习 -> 模拟考试”可以选择全部题库或单个科目、题量和时长。考试只抽取可以自动判分的单选、多选和判断题；作答时不会显示正确答案，交卷后统一显示分数、正确/错误/未答数量和错题回顾。

考试支持暂停、题号总览和退出后继续。未完成考试保存在本机 `quizapp_active_exam`，最近 30 次结果保存在 `quizapp_exam_history`。历史记录可查看全部题目或只看错题/未答，保留用户答案、正确答案和轻量文本快照；旧版只保存错题的记录仍可打开。题库发生删除或覆盖，导致部分题目唯一键无法解析时，应用优先使用当前题库，再回退到历史文本快照。

“学习 -> 统计”除每日时长曲线外，还显示 FSRS 待复习数、错题标注率、错因分布、考试次数和均分。

## 运行与打包

浏览器版可以直接双击 `index.html` 打开。

原生版首页会显示最近一次未完成且已经保存的练习。点击“继续学习”会恢复题库范围、练习模式、题号和已保存答案；设置页可关闭该入口，并分别调整手机和平板宽度。手机宽度范围为 300-430 px，默认 360 px；平板宽度范围为 440-720 px，默认 520 px。关闭退出自动保存后，只有手动点击保存的练习会成为可恢复进度。

原生版保留六套旧版颜色主题，并允许在主题基础上单独选择主色。选择器会检查主色与当前背景的区分度，并自动选择黑色或白色前景保证按钮文字可读；“恢复主题色”会回到当前主题的默认主色。终末地高级主题使用自己的固定配色，不受自定义主色影响。

版本检测不需要单独服务器。应用读取 GitHub Releases Latest，优先比较 release 版本号；当版本号相同但 Release 目标提交与当前 APK 内置构建提交不一致时，也会提示更新。APK 资产名称、大小、digest 和更新时间会参与提醒指纹，避免只按 tag 判断。

- Android APK 内会优先调用系统下载器下载 Release 里的 APK，下载完成后拉起系统安装器。
- 浏览器或网络环境不稳定时，可以打开 GitHub Release 页面手动下载。
- 覆盖安装同包名、同签名的 APK 不会清除本机题库、做题进度、错题集、设置和学习统计；卸载旧版或清除应用数据会删除本地数据。

## Release 题库分发

题库更新同样不需要服务器。设置页里的“检查题库更新”会读取 GitHub Releases Latest，检查期间显示进度条；检查完成后弹窗按“科目 -> 章节”展示可用题库包。用户可以直接勾选整个科目，也可以展开科目后只勾选部分章节；确认“下载题库”后，应用只拉取被勾选的题库文件并写入本机。

1. 优先读取 Release asset 里的 `quizapp-bank-manifest.json`、`QuizApp-bank-manifest.json` 或 `bank-manifest.json`。
2. 新版清单推荐只放题库元信息，每项用 `file` 指向同一个 Release 里的单独题库 JSON；`path` 按 `["科目", "章节"]` 记录逻辑分类。旧版清单直接内嵌完整题库对象仍然兼容。
3. 没有清单时，会自动识别 Release asset 中以 `quizapp-bank-` 开头、以 `.json` 结尾的文件。
4. 拉取到的题库会保存到本机导入题库区，并标记为 Release 分发题库；如果本地已有同路径或同名题库，用户可以选择覆盖，也可以另存为新题库，不会删除做题进度、错题集和其他未选题库。

推荐清单格式：

```json
{
  "banks": [
    {
      "file": "quizapp-bank-001.json",
      "name": "毛概-导论",
      "subject": "毛概",
      "chapter": "导论",
      "path": ["毛概", "导论"],
      "questionCount": 12
    }
  ]
}
```

被引用的题库 JSON 仍使用本项目原有格式，`subject`、`chapter`、`path` 负责分类。以后本项目内置题库推荐使用 `data/科目/章节.json` 物理目录；Release asset 可以继续平铺上传单个题库 JSON，因为分级显示依赖清单和 JSON 内部 `path`，不依赖资产文件夹。Release asset 文件名建议只使用 ASCII，例如 `quizapp-bank-001.json`，避免中文文件名在上传后被平台或命令行工具改写。

APK 默认只内置 `data/` 下的基础题库。`output/xiaoyi-question-banks/export-report.json` 中列出的公开考研图片题库体积较大，由 `scripts/build-release-bank-assets.py` 生成 Release 清单和分题库资产，用户在应用内按科目或章节勾选下载，下载后大型题库保存在 IndexedDB。报告中的 `restrictedBanks` 不会进入 APK 或 Release 资产。只有明确传入 `scripts/build-apk.ps1 -IncludePostgraduateBanks` 时，才会生成包含公开考研图片题库的离线大包。

## Release 公告分发

公告同样不需要服务器。应用启动时会读取 GitHub Releases Latest，如果 Release asset 里存在 `quizapp-announcements.json`，会下载并缓存公告清单；发现未读公告时自动弹窗，首页信件按钮显示红点。没有新公告时自动检测保持静默；只有用户在设置页手动点击“检查公告”时，才提示“没有未读公告”。

推荐公告文件格式：

```json
{
  "announcements": [
    {
      "id": "notice-20260710-1",
      "title": "题库更新说明",
      "date": "2026-07-10",
      "body": "<p>这里写公告正文，可以使用简单 HTML。</p>"
    }
  ]
}
```

`id` 必须稳定且唯一。以后只要在最新版 Release 中替换 `quizapp-announcements.json`，用户下次打开应用就能收到新公告；不需要重新发布 APK。公告正文建议使用简单的 `<p>`、`<ul>`、`<li>`，不要放脚本。

APK 打包：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-apk.ps1
```

Windows EXE 打包：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build-exe.ps1
```

EXE 默认启动本地服务 `http://127.0.0.1:0721/`，题库资源会从内置资源同步到 exe 同级 `data` 文件夹，不依赖用户电脑上的绝对路径。
