# 题库来源与手写学习平台路线图

> 历史调研说明：本文保留题库来源和开源项目调研证据，其中“保留 WebView、不重写 Qt”的旧选型已失效。2026-07-16 起，正式路线为 Qt 6.9.3 + C++17 原生客户端，SpeedyNote 作为固定版本的原生手写核心；以 [综合学习平台实施计划](综合学习平台实施计划.md) 为准。

## 1. 本轮结论

QuizApp 继续采用“本机优先、题库可分发、功能按模块演进”的方向。题库来源数据、题库内置解析和 AI 分析必须分开：

- 题目与来源自带答案/解析属于题库数据。
- 来源解析保存到 `explanations.builtin`，界面固定显示“内置答案与解析”。
- AI 分析仍保存到本机 `quizapp_ai_analyses`，不写回题库，不覆盖内置解析。
- 大型图片题库保存在 IndexedDB；`localStorage` 只保存索引，避免容量上限和重复序列化卡顿。
- 手写笔记保存在 IndexedDB 的独立对象存储，普通做题进度与手写记录互不污染。

公开题库已导出到：

```text
output/xiaoyi-question-banks/
  27严选题基础篇/
  27基础30讲高数/
  export-report.json
  README.md
```

重新导出：

```powershell
node scripts/export-xiaoyi-public-banks.mjs
```

导出器会从来源页面当前前端资源中发现公开 API 配置，只读取无需激活的数据。来源结构变化时也可以通过环境变量提供当前公开 API 地址和公开匿名 Key。

## 3. 题库数据契约

新格式保持对旧 `exp` 字段的兼容，并增加图片题目和结构化来源解析：

```json
{
  "schemaVersion": 2,
  "path": ["考研数学", "题库名", "章节", "小节"],
  "questions": [
    {
      "id": "source:question-id",
      "type": "subjective",
      "q": "第 1 题（题目见图片）",
      "questionImages": ["data:image/png;base64,..."],
      "ans": "",
      "explanations": {
        "builtin": {
          "label": "内置解析",
          "text": "",
          "images": ["data:image/png;base64,..."],
          "videoUrl": "",
          "source": {
            "provider": "xiaoyivip",
            "sourceId": "question-id"
          }
        }
      }
    }
  ]
}
```

支持的题型为 `single`、`multi`、`bool` 和 `subjective`。图片选择题保留 A-D 作答按钮，题目本体在图片中；填空题、解答题和例题按主观题进入手写练题。

## 5. 手写练题工作台

当前实现优先面向宽度不小于 840px 的平板横屏，同时提供手机抽屉式面板：

- 左侧切换题目目录与自由笔记本，中间是分页画布，右侧切换内置解析、AI 分析和图层。
- 钢笔支持真实 pressure、合并高频指针采样和笔迹平滑。
- 防误触默认开启，单指触摸平移，手写笔书写，双指缩放和平移。
- 支持钢笔、荧光笔、笔画橡皮、撤销重做、颜色和笔宽调整。
- 每道题绑定独立文档；页面内支持多图层、文本、图片和题目对象，选择后可以拖动。
- 自由笔记与题目笔记共用格式，保存在 IndexedDB；旧版单页笔迹首次打开时自动迁移。
- 题目图片、正确答案、来源内置解析与 AI 分析各自独立。

## 6. 综合学习平台路线

### 阶段 A：数据底座

1. 完成题库 schema 版本、来源信息、哈希、更新时间和兼容迁移。
2. 题库分发改为清单、按章资源和增量下载，安装包不内嵌大型可选题库。
3. 增加题库完整性检查：题量、答案缺失、图片损坏、重复题、异常题型。
4. 增加本机存储用量页面，允许按题库清理缓存，不影响错题和进度。

### 阶段 B：学习工作台

1. 手写模式增加套索、局部擦除、缩放和平移、纸张模板、横竖纸张。
2. 增加 PDF 导入与逐页批注，笔记可以关联科目、章节和题目。
3. 增加闪卡、知识点笔记、每日复习队列和基于遗忘曲线的复习计划。
4. 增加考试模拟、限时、暂停、交卷、成绩趋势和错因标签。

### 阶段 C：智能学习

1. AI 分析始终以题库给定答案为事实输入，并显式显示模型和时间。
2. 图片题仅在用户启用视觉模型时发送图片；不支持图片时明确提示。
3. 增加 AI 生成相似题、知识点问答、错题归因和复习摘要，所有生成内容标记“AI”。
4. 增加可选 OCR 管线，把图片题转换为可搜索文本，但保留原图作为权威来源。

### 阶段 D：客户端与同步

1. 先把当前单文件逻辑拆成数据、题库、练习、手写、AI、发布六个模块，再引入服务端。
2. 服务端只同步用户主动选择的数据；API Key 默认只保存在设备，不上传。
3. 账号、通知、题库、版本和公告使用独立资源版本，不用应用版本号代替题库版本。
4. 支持本地导出完整备份，覆盖安装保持包名、签名和本机数据库不变。

### 阶段 E：质量与可访问性

1. 手机竖屏、平板横屏、桌面三套回归矩阵，覆盖触摸、手写笔、鼠标和系统返回键。
2. 题库导入、更新、覆盖、另存、回收站、恢复均提供事务和失败回滚。
3. 大题库采用虚拟列表、按需解码图片和后台持久化，不在主线程反复 stringify 全量数据。
4. 所有外部内容显示来源、授权边界和更新时间；受限内容不绕过访问控制。

## 7. 架构约束

- “功能多”不等于把代码继续堆进 `index.html`。下一阶段必须先模块化，再增加 PDF、闪卡或云同步。
- APK 和 EXE 的基础安装包保持轻量，大型题库通过 Release 资源或用户导入按需安装。
- iPad 无法安装 Android APK；Web/PWA 或独立 iOS 包需要单独签名与发布链路。当前 Web 手写工作台可在 Safari 技术验证，但不能用 Android 构建冒充 iPad 原生支持。
- 外部题库能否公开再分发由内容授权决定。导出成功只证明技术可读，不自动获得再发布权。
